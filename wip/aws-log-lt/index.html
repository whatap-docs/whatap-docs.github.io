라이브 Tail

홈 화면 > 프로젝트 선택 > 로그 > 라이브 Tail

와탭 모니터링 서비스 초기 화면에서 프로젝트를 선택한 다음 프로젝트 메뉴 하위에 로그 > 라이브 Tail 메뉴를 선택하세요.

라이브 Tail 메뉴에서 복잡한 로그들도 손쉽게 접근 가능합니다. 필요에 따라 필터링 혹은 하이라이트 등의 기능을 활용해 실시간으로 조회할 수 있습니다. 로그 데이터 조회 주기는 2초입니다. 주요 용어는 다음과 같습니다.

  • Category: 로그의 수집 및 조회 단위입니다.
  • Content: 로그 메시지입니다.
  • Search Key: 로그 Parser 설정을 통해 생성합니다.
  • Tag: 수집된 로그를 검색할 수 있는 검색 키입니다.

필터 영역​

  • 필터 적용
    필터를 적용하면 입력한 조건에 맞는 로그를 필터링합니다. 복수의 필터를 입력할 수 있습니다. 필터의 태그가 같은 경우 OR(||)로, 그렇지 않은 경우는 AND(&&)로 적용됩니다. 입력창에 값을 직접 입력하거나 입력창을 클릭하여 필터를 선택할 수 있습니다. 검색 버튼을 클릭하세요. 영역에서 조회할 수 있습니다.

  • 필터 수정
    필터에 값을 입력한 뒤 입력한 값을 클릭하면 해당 값을 수정할 수 있습니다.

  • 필터 수식
    입력된 필터 값 아래에 있는 수식(expression)은 로그 데이터 조회 시 적용될 필터 수식 미리 보기입니다.

  • 검색 키(Search Key)
    다음 이미지에서 파란색 박스 부분은 파싱(parsing)된 검색 키입니다.

Search Key

검색 키는 로그 > 설정의 로그 Parser 설정에서 파싱 로직을 등록해 설정할 수 있습니다.

  • 필터 입력 문법
    필터 입력 시 사용할 수 있는 문법 목록입니다. 라이브 Tail과 로그 검색기는 동일한 문법을 사용합니다.

    문법예시설명
    include(+)+ote-gatewayote-gateway를 포함한 것만 필터링
    exclude(-)-ote-gatewayote-gateway를 제외한 것만 필터링
    startWith(*)ERROR*ERROR로 시작하는 것만 필터링
    endWith(*)*ERRORERROR로 끝나는 것만 필터링

콘텐츠 필터 영역​

  • Content 필터
    로그의 콘텐츠 중 해당 키워드를 포함한 로그만 필터링 합니다.

    • 입력창에 필터를 입력하고 검색 버튼을 클릭하세요. 클라이언트에서 로그 데이터를 필터링하고 영역에서 조회할 수 있습니다.
    • 키워드를 포함한 Content의 문자열은 하이라이팅 됩니다.
    • 단일 또는 복수 키워드로 필터를 걸 수 있습니다.

  • Content 필터 조건
    복수 키워드로 필터를 할 경우 다음과 같이 작성합니다.

    입력 문자열설명결과
    a b c띄어쓰기로 각 키워드를 구분합니다.a, b, c
    "Whatap is good."띄워쓰기를 키워드에 포함하고 싶은 경우 '' 또는 ""로 감쌉니다.Whatap is good.
    "Whatap\ is good."""로 감싸진 키워드에서 \를 포함할 경우, \로 입력해야합니다.Whatap\ is good.

  • 하이라이트
    원하는 키워드를 손쉽게 식별하기 위해서 하이라이트 기능을 제공합니다.

    • 하이라이트 버튼을 클릭하여 키워드를 입력하세요. 입력창 왼쪽 색상을 클릭하면 선택할 수 있는 색상 메뉴가 나타납니다.
    • 기본적으로 로그 레벨에 따른 하이라이팅(WARN, ERROR, FATAL)이 적용되어 있습니다.
    • 설정한 내용은 프로젝트 단위로 저장됩니다.

  • 테이블 설정
    테이블 설정 메뉴는 라이브 Tail, 검색기, 탐색기에서 사용할 수 있습니다.


    영역 왼쪽의 테이블 설정 버튼을 클릭하면 컬럼 추가와 테이블 컨텐츠 설정 옵션 메뉴가 나타납니다.

    1. 컬럼
    • 컬럼 추가: 태그를 선택하여 테이블에 컬럼을 추가할 수 있습니다.
    • 컬럼 순서 설정: 컬럼을 추가하면 컬럼 순서 설정에 해당 컬럼이 추가됩니다. 원하는 컬럼을 드래그하여 컬럼의 순서를 변경하세요.
    1. 테이블
    • 컨텐츠 표시 여부
      • 체크된 항목은 테이블에 표시되지 않습니다. 기본으로 content, tag 모두 체크가 되어있으며 두 가지 항목 모두 표시합니다.
      • 다음과 같이 tag를 해제할 경우, 테이블에서 로그의 tag는 표시되지 않습니다.
    • 태그 관리
      • 태그 관리 목록에 태그를 추가하면 추가한 순서대로 로그의 태그가 나열됩니다. 태그의 순서는 드래그하여 변경할 수 있습니다.
      • 추가한 태그를 비활성화하면 비활성화한 태그는 로그의 태그에 노출되지 않습니다.
