MySQL RDS 지표

데이터베이스 메트릭은 데이터베이스의 작업량과 리소스 사용량을 포괄적으로 수집합니다. 와탭 모니터링 서비스는 지속적으로 메트릭을 수집하여 데이터베이스의 병목 현상을 확인하고, 성능 문제의 근본 원인을 식별하는 데 도움이 되는 데이터를 제공합니다.

MySQL 메트릭​

지표 이름설명
Aborted_connectsMySQL서버에 연결하기 위해 시도한 실패 수
active_sessions쿼리를 실행중인 세션 수. Threads_running과 같음
Connection_errors_internal내부 서버 오류, 예를 들어 메모리 오류가 발생하지 않거나 실패 스레드가 시작됩니다.글로벌 값은 플러시 상태로 플러시 될 수 있습니다.
Connection_errors_max_connectionsmax_Connections 한계에 도달하여 연결된 연결 수.글로벌 값은 플러시 상태로 플러시 될 수 있습니다.
ConnectionsDB 에 접속 시도 횟수
Threads_connected현재 접속 되어 있는 세션 갯수
max_connections최대 동시 클라이언트 연결 수. 커넥션 수가 부족할 경우 'Too ManyConnection Error'가 발생할 수 있습니다.
Bytes_receiveddb서버가 받은 데이터 bytes. Bytes_received 가 높다면 이 서버는 쓰기 위주작업(insert 등) 서버임을 알 수 있습니다.
Bytes_sentdb 서버가 보낸 데이터 bytes. Bytes_sent가 높다면 이 서버는 읽기위주 작업(select) 서버임을 알 수 있습니다. Questions, Uptime과 함께 응용해보면 1개 쿼리당 평균 Byte, 시간당 처리 Byte등을 계산 할 수 있습니다.
Innodb_buffer_pool_bytes_dirty버퍼풀의 데이터 중 변경된 바이트 수(더티 데이터 수)
Innodb_buffer_pool_pages_data전체 버퍼풀에서 현재 사용중인 버퍼풀 페이지 수
Innodb_buffer_pool_pages_dirty버퍼풀의 데이터 중 변경된 페이지 수(더티 페이지 수)
Innodb_buffer_pool_pages_flushed플러시 된 InnoDB 버퍼풀 페이지 수. 이는 버퍼풀의 더티 데이터를 디스크에 쓰는 과정으로 과도한 flush는 디스크에 부담을 줄 수 있으므로 디스크의 성능과 함께 모니터링 해야 합니다.
Innodb_buffer_pool_pages_free무료 innodb 버퍼 풀 페이지 수.
Innodb_buffer_pool_pages_total전체 버퍼풀의 페이지 수
Innodb_buffer_pool_read_requests논리적 읽기 요청 수(물리적 읽기를 포함). Inodb buffer pool hit율 계산을 다음과 같이 할 수 있으며, 90% 이상을 권장합니다. 만약 Hit Ratio가 낮다면 Innodb_buffer_pool_size증설을 검토합니다.
Buffer Hit Ratio = (1 - (Innodb_buffer_pool_reads/Innodb_buffer_pool_read_requests)) * 100
Innodb_buffer_pool_reads물리적(Disk) read 를 한 횟수. Innodb 버퍼풀에 없어서 디스크에서 직접읽어야 하는 논리적 읽기 요청 수
Binlog_bytes_written이진 로그에 기록 된 바이트 수.
Binlog_commits이진 로그에 약속 된 총 거래 수.
Innodb_row_lock_timerow lock 획득에 소비된 총 시간 (milliseconds)
Innodb_row_lock_waitsInnodb 스토리지 엔진 테이블에서 row lock을 기다려야 하는 횟수. Lock을 획득한 수가 아닌 lock을 얻기 위해 대기하는 횟수로 이 지표가 평소보다 급증한다면 쿼리 대기가 발생(락 경합)하는 것으로 쿼리 튜닝이 필요합니다.
lock_wait_sessions현재 row lock을 획득하기 위해 대기중인 수. Innodb_row_lock_current_waits과 같음
replication_delayReplication구조일 경우 replication db에 반영되는 지연시간. Slave DB에서 측정되는 지표입니다.
Slave_heartbeat_period하트 비트 패킷이 노예가 마스터로부터 요청하는 초의 시간.
Slaves_running실행되는 슬레이브 SQL 스레드 수.
Slow_querieslong_query_time 초 이상 걸린 쿼리 수
Sort_rowssort 처리한 row 수
Com_commitcommit 횟수
Com_delete실행된 삭제 명령 수 handler_delete와는 다른 행이 테이블에서 삭제 된 횟수를 계산합니다.
Com_insert실행된 삽입 명령 수

Com_selectSelect 명령을 실행한 횟수

Innodb_rows_readselect 에서 처리된 row 수

Writeswrite 에 사용한 명령 횟수. Com_insert + Com_update + Com_delete 와 같음
Aborted_clients정상적으로 종료되지 않은 클라이언트로 인해 중된된 연결 수.

Connection_errors_peer_address연결 클라이언트 IP 주소를 검색하는 동안 오류 수글로벌 값은 플러시 상태로 플러시 될 수 있습니다.

Delayed_errors삽입 중에 발생한 오류 수는 지연되었습니다. 글로벌 값은 플러시 상태로 플러시 될 수 있습니다.
Delayed_insert_threads지연된 스레드 삽입 수.
Delayed_writes삽입 지연 행 수. 글로벌 값은 플러시 상태로 플러시 될 수 있습니다.

Empty_queries결과가없는 쿼리 수. 이것은 com_empty_query와 같지 않습니다.
Executed_eventsCreate Event와 함께 생성 된 이벤트가 실행되었습니다. 이는 이벤트가 실행될 때만 증가 할 때만 증가한다는 점에서 com_create_event와 다릅니다.
Executed_triggersCreate Trigger와 함께 생성 된 트리거가 횟수로 실행되었습니다. 이는 트리거가 실행될 때만 발생할 때만 증가한다는 점에서 com_create_trigger와 다릅니다.
Feature_check_constraint제약 조건 수가 확인되었습니다.글로벌 값은 플러시 상태로 플러시 될 수 있습니다.
Feature_custom_aggregate_functions사용자 정의 집계 기능을 사용하는 쿼리 수.
Feature_delay_key_writeDelay_key_write를 사용하는 테이블 수입니다.글로벌 값은 플러시 상태로 플러시 될 수 있습니다.
Feature_dynamic_columnscolumn_create () 함수가 사용 된 횟수입니다.

Innodb_buffer_pool_pages_miscRow lock, hash index와 같이 오버헤드에 할당되어 사용된 버퍼풀의 페이지 수

Innodb_buffer_pool_read_ahead미리읽기 백그라운드 스레드가 버퍼풀로 미리 읽어들인 페이지 수

Innodb_buffer_pool_read_ahead_rnd랜덤으로 미리 읽기가 발생한 페이지 수
